## üìù Content Rules

### SEO Requirements
1. **Keyword Research:** Each post must target 1-2 primary keywords
2. **Word Count:** Minimum 1,500 words for comprehensive guides
3. **Structure:** Use H2, H3 headings for scannable content
4. **Internal Links:** Link to at least 2 related articles in each post
5. **Meta Description:** Every post needs unique meta description (150-160 chars)

### Quality Standards
1. **Accuracy:** Verify all tax/legal information against official IRS sources
2. **Citations:** Include links to authoritative sources (IRS, blockchain docs)
3. **Actionable:** Each guide should have clear next steps for readers
4. **Original:** No plagiarism; add unique insights and examples
5. **Updated:** Review and update tax posts annually for law changes

### Content Tone
- **Educational, not advisory** ‚Äî Say "The IRS requires..." not "You must..."
- **Professional but accessible** ‚Äî Complex topics explained simply
- **Arthur Labs attribution** ‚Äî Include relevant ecosystem tool references where natural
- **Disclaimer** ‚Äî Tax posts must include "consult a qualified tax professional" disclaimer

### Technical Requirements
1. **MDX Format:** Use `.mdx` files in `/src/app/blogs/[slug]/`
2. **Frontmatter:** Include title, description, date, author, tags
3. **Images:** Optimize to WebP, include alt text
4. **Code Blocks:** Use for technical tutorials with syntax highlighting
